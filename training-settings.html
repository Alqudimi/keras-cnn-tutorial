الرئيسية إعدادات التدريب

إعدادات التدريب الأساسية والمتقدمة

لتحقيق أفضل أداء من نموذج التعلم العميق، لا يكفي تصميم بنية شبكة جيدة فحسب، بل يجب أيضًا ضبط إعدادات التدريب (Hyperparameters) بعناية. هذا الدليل يشرح أهم هذه الإعدادات وكيفية تطبيقها.

المقدمة

المعاملات الفائقة (Hyperparameters) هي الإعدادات التي نحددها قبل بدء التدريب والتي تتحكم في كيفية تعلم النموذج. على عكس الأوزان التي يتعلمها النموذج تلقائيًا، المعاملات الفائقة يجب أن نختارها نحن بعناية.

المحاور الستة الرئيسية

  • المُحسّنات (Optimizers): خوارزميات تحديث الأوزان
  • مجدولات معدل التعلم: تغيير معدل التعلم أثناء التدريب
  • الإيقاف المبكر: منع الإفراط في التكييف
  • تنظيم L2: تضاؤل الأوزان
  • تنعيم التسميات: تقليل الثقة المفرطة
  • أوزان الفئات: معالجة البيانات غير المتوازنة

اختيار وضبط المُحسّن (Optimizer)

المُحسّن هو الخوارزمية التي تعدل أوزان الشبكة العصبية لتقليل دالة الخسارة. اختيار المُحسّن المناسب وضبط معاملاته يؤثر بشكل كبير على سرعة وجودة التدريب.

المُحسّنات الشائعة

المُحسّن الوصف المعاملات الأساسية متى تستخدم؟
Adam الأكثر شيوعًا، يجمع بين Adagrad و RMSprop lr=0.001, β1=0.9, β2=0.999 الخيار الافتراضي لمعظم المهام
RMSprop يستخدم متوسط مربع التدرجات lr=0.001, ρ=0.9 للبيانات غير الثابتة أو RNN
SGD الخوارزمية الأساسية مع momentum lr=0.01, momentum=0.9 للحصول على أفضل أداء نهائي
AdamW Adam مع تضاؤل أوزان صحيح lr=0.001, weight_decay=0.01 عند استخدام تنظيم L2
Python 
from tensorflow.keras.optimizers import Adam, SGD, RMSprop

# ═══════════════════════════════════════════════════
# Adam: الخيار الافتراضي والأكثر شيوعًا
# ═══════════════════════════════════════════════════
optimizer_adam = Adam(
    learning_rate=0.001,  # معدل التعلم
    beta_1=0.9,           # معدل التسوس للتقديرات الأولى
    beta_2=0.999,         # معدل التسوس للتقديرات الثانية
    epsilon=1e-7          # عامل صغير لمنع القسمة على صفر
)

# ═══════════════════════════════════════════════════
# SGD مع Momentum و Nesterov
# ═══════════════════════════════════════════════════
optimizer_sgd = SGD(
    learning_rate=0.01,   # معدل تعلم أعلى من Adam
    momentum=0.9,         # للتغلب على المناطق المسطحة
    nesterov=True         # تسريع التقارب
)

# ═══════════════════════════════════════════════════
# تجميع النموذج
# ═══════════════════════════════════════════════════
model.compile(
    optimizer=optimizer_adam,
    loss='categorical_crossentropy',
    metrics=['accuracy']
)

print("✅ تم إعداد المُحسّن")

نصيحة: استراتيجية الاختيار

ابدأ بـ Adam للتقارب السريع. إذا أردت الحصول على أفضل أداء ممكن، انتقل إلى SGD مع momentum بعد أن يستقر التدريب. SGD قد يحقق نتائج أفضل في النهاية لكنه أبطأ وأكثر حساسية للمعاملات.

مجدول معدل التعلم (Learning Rate Scheduler)

بدلاً من استخدام معدل تعلم ثابت طوال التدريب، يقوم المجدول بتغيير معدل التعلم تدريجيًا. هذا يمنع النموذج من "القفز" فوق الحد الأدنى للخسارة في المراحل المتأخرة ويسمح بتقارب أفضل.

التقنيات الشائعة

التقنية الوصف الاستخدام الأمثل
Exponential Decay يقلل معدل التعلم بشكل أسي تقليل تدريجي مستمر
Step Decay يقلل بنسبة ثابتة كل N خطوات تحكم دقيق في نقاط التقليل
Cosine Annealing يتبع منحنى جيب التمام تقليل سلس مع إعادة التسخين
ReduceLROnPlateau يقلل عند توقف التحسن تكيفي ومرن
Python 
from tensorflow.keras.optimizers import Adam
from tensorflow.keras.optimizers.schedules import ExponentialDecay
from tensorflow.keras.callbacks import ReduceLROnPlateau

# ═══════════════════════════════════════════════════
# الطريقة 1: Exponential Decay
# ═══════════════════════════════════════════════════
initial_learning_rate = 0.01

lr_schedule = ExponentialDecay(
    initial_learning_rate,
    decay_steps=10000,      # عدد الخطوات قبل التقليل
    decay_rate=0.9,         # معدل التقليل (90% من القيمة السابقة)
    staircase=True          # تقليل متقطع (درجات)
)

# تمرير المجدول للمُحسّن
optimizer = Adam(learning_rate=lr_schedule)
model.compile(optimizer=optimizer, loss='categorical_crossentropy')

# ═══════════════════════════════════════════════════
# الطريقة 2: ReduceLROnPlateau (Callback)
# يقلل تلقائيًا عند توقف التحسن
# ═══════════════════════════════════════════════════
reduce_lr = ReduceLROnPlateau(
    monitor='val_loss',     # المقياس المراقب
    factor=0.5,             # يصبح النصف عند التقليل
    patience=5,             # انتظار 5 حقبات قبل التقليل
    min_lr=1e-7,            # الحد الأدنى لمعدل التعلم
    verbose=1               # طباعة رسالة عند التقليل
)

# استخدام في التدريب
history = model.fit(
    x_train, y_train,
    validation_data=(x_val, y_val),
    epochs=100,
    callbacks=[reduce_lr]
)

الإيقاف المبكر (Early Stopping)

الإيقاف المبكر هو شكل من أشكال التنظيم (Regularization) يوقف التدريب عندما يبدأ أداء النموذج في التدهور على مجموعة التحقق. هذا يمنع الإفراط في التكييف (Overfitting) ويوفر الوقت والموارد.

كيف يعمل؟

يراقب الإيقاف المبكر مقياسًا معينًا (عادة خسارة التحقق). إذا لم يتحسن هذا المقياس لعدد معين من الحقبات (patience)، يتوقف التدريب ويستعيد أفضل أوزان تم الوصول إليها.

المعامل الوصف القيمة الموصى بها
monitor المقياس الذي يجب مراقبته 'val_loss'
patience عدد الحقبات قبل الإيقاف 5 - 15
mode اتجاه التحسن 'min' للخسارة، 'max' للدقة
restore_best_weights استعادة أفضل أوزان True (موصى به جدًا)
min_delta الحد الأدنى للتحسن المعتبر 0.0001 - 0.001
Python 
from tensorflow.keras.callbacks import EarlyStopping

# ═══════════════════════════════════════════════════
# إعداد الإيقاف المبكر
# ═══════════════════════════════════════════════════
early_stopping = EarlyStopping(
    monitor='val_loss',           # مراقبة خسارة التحقق
    patience=10,                  # انتظار 10 حقبات
    mode='min',                   # نريد تقليل الخسارة
    restore_best_weights=True,    # استعادة أفضل أوزان
    min_delta=0.0001,             # الحد الأدنى للتحسن
    verbose=1                     # طباعة رسالة عند الإيقاف
)

# ═══════════════════════════════════════════════════
# التدريب مع الإيقاف المبكر
# ═══════════════════════════════════════════════════
history = model.fit(
    x_train, y_train,
    validation_data=(x_val, y_val),
    epochs=100,                   # عدد كبير من الحقبات
    callbacks=[early_stopping]    # سيتوقف مبكرًا إذا لزم الأمر
)

# التحقق من عدد الحقبات الفعلية
actual_epochs = len(history.history['loss'])
print(f"توقف التدريب عند الحقبة: {actual_epochs}")

نصيحة مهمة

دائمًا استخدم restore_best_weights=True! بدونه، ستحصل على أوزان آخر حقبة وليس أفضل حقبة. هذا قد يعني فقدان عدة نقاط مئوية من الدقة.

تضاؤل الأوزان / تنظيم L2 (Weight Decay)

تضاؤل الأوزان (أو تنظيم L2) هو تقنية تنظيمية تضيف عقوبة إلى دالة الخسارة تتناسب مع مربع حجم الأوزان. هذا يجبر النموذج على استخدام أوزان أصغر وأكثر انتشارًا، مما يقلل من تعقيده ويمنع الإفراط في التكييف.

الفكرة الرياضية

دالة الخسارة الجديدة = الخسارة الأصلية + λ × Σ(w²)
حيث λ هو معامل التنظيم و w هي الأوزان.

Python 
from tensorflow.keras import layers, regularizers

# ═══════════════════════════════════════════════════
# معامل التنظيم L2
# ═══════════════════════════════════════════════════
L2_FACTOR = 0.001  # قيم شائعة: 0.0001 - 0.01

# ═══════════════════════════════════════════════════
# تطبيق L2 على طبقات الالتفاف
# ═══════════════════════════════════════════════════
model.add(layers.Conv2D(
    32, (3, 3),
    padding='same',
    kernel_regularizer=regularizers.l2(L2_FACTOR),  # L2 على الأوزان
    bias_regularizer=regularizers.l2(L2_FACTOR)     # اختياري: L2 على الانحياز
))

# ═══════════════════════════════════════════════════
# تطبيق L2 على الطبقات الكثيفة
# ═══════════════════════════════════════════════════
model.add(layers.Dense(
    128,
    activation='relu',
    kernel_regularizer=regularizers.l2(L2_FACTOR)
))

# ═══════════════════════════════════════════════════
# الجمع بين L1 و L2 (Elastic Net)
# ═══════════════════════════════════════════════════
elastic_reg = regularizers.l1_l2(l1=0.0001, l2=0.001)
model.add(layers.Dense(64, kernel_regularizer=elastic_reg))

print("✅ تم تطبيق تنظيم L2")

تنعيم التسميات (Label Smoothing)

تنعيم التسميات هو تقنية تنظيمية تقلل من ثقة النموذج المفرطة في التسميات "الصعبة" (Hard Labels). بدلاً من التسمية [0, 0, 1, 0] نستخدم [0.025, 0.025, 0.925, 0.025].

الفوائد

  • تقليل الثقة المفرطة: يمنع النموذج من أن يصبح "متعصبًا"
  • تحسين المعايرة: احتمالات النموذج تصبح أكثر دقة
  • تنظيم ضمني: يحسن قدرة التعميم
  • التعامل مع الضوضاء: أفضل للبيانات ذات التسميات غير الدقيقة
Python 
from tensorflow.keras.losses import CategoricalCrossentropy

# ═══════════════════════════════════════════════════
# معامل التنعيم
# ═══════════════════════════════════════════════════
SMOOTHING_FACTOR = 0.1  # قيم شائعة: 0.1 - 0.2

# ═══════════════════════════════════════════════════
# دالة الخسارة مع تنعيم التسميات
# ═══════════════════════════════════════════════════
loss_fn = CategoricalCrossentropy(
    from_logits=False,            # إذا كانت المخرجات احتمالات (softmax)
    label_smoothing=SMOOTHING_FACTOR
)

# تجميع النموذج
model.compile(
    optimizer='adam',
    loss=loss_fn,
    metrics=['accuracy']
)

# ═══════════════════════════════════════════════════
# كيف يتم التنعيم (للفهم):
# ═══════════════════════════════════════════════════
# التسمية الأصلية: [0, 0, 1, 0, 0]
# بعد التنعيم (ε=0.1):
# - القيمة 1 تصبح: 1 - ε = 0.9
# - القيم 0 تصبح: ε / (num_classes - 1) = 0.1/4 = 0.025
# النتيجة: [0.025, 0.025, 0.9, 0.025, 0.025]

print("✅ تم إعداد تنعيم التسميات")

معالجة الفئات غير المتوازنة (Class Weighting)

عندما يكون عدد الأمثلة في فئة ما أكبر بكثير من فئة أخرى (Imbalanced Dataset)، يميل النموذج إلى الانحياز للفئة الأكثر تكرارًا. وزن الفئات يعطي أهمية أكبر للفئات النادرة.

حساب أوزان الفئات

الصيغة: وزن الفئة = إجمالي العينات / (عدد الفئات × عدد عينات الفئة)

Python 
import numpy as np
from sklearn.utils.class_weight import compute_class_weight

# ═══════════════════════════════════════════════════
# مثال: بيانات غير متوازنة
# الفئة A: 1000 صورة
# الفئة B: 100 صورة
# الفئة C: 50 صورة
# ═══════════════════════════════════════════════════

# الطريقة 1: الحساب اليدوي
total_samples = 1150
num_classes = 3
class_counts = {0: 1000, 1: 100, 2: 50}

class_weights = {}
for class_idx, count in class_counts.items():
    weight = total_samples / (num_classes * count)
    class_weights[class_idx] = weight

print(f"أوزان الفئات: {class_weights}")
# {0: 0.38, 1: 3.83, 2: 7.67}

# ═══════════════════════════════════════════════════
# الطريقة 2: استخدام sklearn
# ═══════════════════════════════════════════════════
y_train_labels = np.array([0]*1000 + [1]*100 + [2]*50)

class_weights_sklearn = compute_class_weight(
    class_weight='balanced',
    classes=np.unique(y_train_labels),
    y=y_train_labels
)

class_weights_dict = dict(enumerate(class_weights_sklearn))
print(f"أوزان sklearn: {class_weights_dict}")

# ═══════════════════════════════════════════════════
# استخدام في التدريب
# ═══════════════════════════════════════════════════
history = model.fit(
    x_train, y_train,
    validation_data=(x_val, y_val),
    epochs=50,
    class_weight=class_weights_dict  # تمرير الأوزان هنا
)

تحذير: الأوزان العالية جدًا

إذا كان عدم التوازن شديدًا جدًا (مثلاً 100:1)، قد تكون الأوزان المحسوبة كبيرة جدًا. في هذه الحالة، جرب تقليل الأوزان قليلاً أو استخدام تقنيات أخرى مثل Oversampling أو SMOTE.

أفضل الممارسات والنصائح

ابدأ دائمًا بالإعدادات الافتراضية: Adam optimizer مع lr=0.001، batch_size=32، بدون تنظيم L2. راقب الأداء على مجموعة التحقق، ثم عدّل الإعدادات واحدة تلو الأخرى.

إذا كانت خسارة التدريب تنخفض بينما خسارة التحقق ترتفع: Overfitting → أضف تنظيم (Dropout, L2, Data Augmentation). إذا كانت كلتاهما عالية: Underfitting → زد قدرة النموذج أو درّب لفترة أطول.

احفظ أفضل نموذج تلقائيًا باستخدام ModelCheckpoint مع save_best_only=True. هذا يضمن عدم فقدان أفضل أداء تم الوصول إليه حتى لو تدهور الأداء لاحقًا.

جرب استراتيجية التدريب على ثلاث مراحل بمعدلات تعلم مختلفة: مرحلة أولى بمعدل عالٍ (1e-3)، مرحلة ثانية بمعدل متوسط (1e-4)، ومرحلة ثالثة بمعدل منخفض (1e-5). هذا موضح بالتفصيل في صفحة مراحل التدريب.
