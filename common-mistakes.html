الأخطاء الشائعة (Common Mistakes)

تعرف على أكثر الأخطاء شيوعاً في التعلم العميق وكيفية تجنبها وإصلاحها.

مقدمة

حتى المحترفون يقعون في هذه الأخطاء. الفرق هو أن المحترف يعرف كيف يتعرف عليها ويصلحها بسرعة. هذا الدليل سيساعدك على تجنب أسابيع من الإحباط!

نصيحة ذهبية

قبل البحث عن الأخطاء المعقدة، تأكد من الأساسيات: هل البيانات محملة بشكل صحيح؟ هل التسميات متطابقة؟ هل هناك تسرب للبيانات (Data Leakage)؟

Overfitting (الإفراط في التكييف)

ما هو؟

النموذج يحفظ بيانات التدريب بدلاً من تعلم الأنماط العامة. يؤدي إلى أداء ممتاز على التدريب لكن سيئ على البيانات الجديدة.

كيف تتعرف عليه؟

العلامات 
علامات Overfitting:

1. Training accuracy عالية جداً (>98%) بينما Validation منخفضة
2. Training loss تنخفض لكن Validation loss تبدأ بالارتفاع
3. فجوة كبيرة بين أداء التدريب والتحقق

مثال نموذجي:
Epoch 50: train_acc=99.5%, val_acc=72.3%  ← فجوة 27%!

Underfitting (نقص التكييف)

ما هو؟

النموذج بسيط جداً لاستيعاب تعقيد البيانات. الأداء سيئ على التدريب والتحقق معاً.

كيف تتعرف عليه؟

العلامات 
علامات Underfitting:

1. Training accuracy منخفضة (مثلاً 60%)
2. Validation accuracy منخفضة أيضاً وقريبة من Training
3. الـ Loss لا ينخفض بشكل ملحوظ
4. المنحنيات مسطحة

مثال نموذجي:
Epoch 50: train_acc=62.1%, val_acc=60.5%  ← كلاهما منخفض!

Learning Rate خارج السيطرة

مشاكل LR المرتفع

العلامات والحلول 
علامات LR مرتفع جداً:
- Loss يتذبذب بشدة أو ينفجر (NaN)
- Accuracy تقفز صعوداً وهبوطاً
- التدريب غير مستقر

مثال:
Epoch 1: loss=2.5, acc=35%
Epoch 2: loss=nan, acc=10%  ← انفجار!

# أو:
Epoch 1: loss=1.2, acc=65%
Epoch 2: loss=3.4, acc=25%  ← تذبذب!
Epoch 3: loss=0.8, acc=72%
Epoch 4: loss=2.1, acc=40%

مشاكل LR المنخفض

العلامات والحلول 
علامات LR منخفض جداً:
- Loss ينخفض ببطء شديد
- يحتاج مئات الـ Epochs للتحسن
- يعلق في local minimum

مثال:
Epoch 1:   loss=2.5, acc=35%
Epoch 50:  loss=2.4, acc=37%  ← تحسن ضئيل جداً!
Epoch 100: loss=2.3, acc=39%

# الحل 1: Learning Rate Finder
def find_learning_rate(model, train_data, start_lr=1e-7, end_lr=1):
    """البحث عن أفضل learning rate"""
    import math
    
    num_batches = len(train_data)
    lr_multiplier = (end_lr / start_lr) ** (1 / num_batches)
    
    lrs = []
    losses = []
    
    model_copy = tf.keras.models.clone_model(model)
    model_copy.set_weights(model.get_weights())
    
    optimizer = tf.keras.optimizers.Adam(learning_rate=start_lr)
    model_copy.compile(optimizer=optimizer, loss='sparse_categorical_crossentropy')
    
    lr = start_lr
    for batch, (x, y) in enumerate(train_data):
        with tf.GradientTape() as tape:
            pred = model_copy(x, training=True)
            loss = tf.keras.losses.sparse_categorical_crossentropy(y, pred)
            loss = tf.reduce_mean(loss)
        
        grads = tape.gradient(loss, model_copy.trainable_variables)
        optimizer.apply_gradients(zip(grads, model_copy.trainable_variables))
        
        lrs.append(lr)
        losses.append(float(loss))
        
        lr *= lr_multiplier
        optimizer.learning_rate.assign(lr)
        
        if loss > 10 or math.isnan(loss):
            break
    
    # رسم النتائج
    plt.figure(figsize=(10, 6))
    plt.plot(lrs, losses)
    plt.xscale('log')
    plt.xlabel('Learning Rate')
    plt.ylabel('Loss')
    plt.title('Learning Rate Finder')
    plt.show()
    
    # أفضل LR هو حيث يبدأ الانحدار الحاد
    return lrs, losses

# الحل 4: Warmup
class WarmUpSchedule(tf.keras.optimizers.schedules.LearningRateSchedule):
    def __init__(self, initial_lr, warmup_steps, decay_schedule):
        self.initial_lr = initial_lr
        self.warmup_steps = warmup_steps
        self.decay_schedule = decay_schedule
    
    def __call__(self, step):
        warmup_lr = self.initial_lr * (step / self.warmup_steps)
        decay_lr = self.decay_schedule(step - self.warmup_steps)
        return tf.cond(step < self.warmup_steps, lambda: warmup_lr, lambda: decay_lr)

# كشف عدم التوازن
def check_class_balance(dataset):
    class_counts = {}
    for _, labels in dataset:
        for label in labels.numpy():
            class_counts[label] = class_counts.get(label, 0) + 1
    
    total = sum(class_counts.values())
    print("Class Distribution:")
    for cls, count in sorted(class_counts.items()):
        ratio = count / total * 100
        print(f"  Class {cls}: {count} ({ratio:.1f}%)")
    
    imbalance_ratio = max(class_counts.values()) / min(class_counts.values())
    print(f"\nImbalance Ratio: {imbalance_ratio:.1f}:1")
    
    if imbalance_ratio > 3:
        print("⚠️ Warning: Significant class imbalance detected!")

# الحل 3: Focal Loss (للحالات الشديدة)
def focal_loss(gamma=2.0, alpha=0.25):
    def focal_loss_fn(y_true, y_pred):
        epsilon = tf.keras.backend.epsilon()
        y_pred = tf.clip_by_value(y_pred, epsilon, 1.0 - epsilon)
        
        cross_entropy = -y_true * tf.math.log(y_pred)
        weight = alpha * tf.pow(1 - y_pred, gamma)
        focal = weight * cross_entropy
        
        return tf.reduce_mean(tf.reduce_sum(focal, axis=-1))
    
    return focal_loss_fn
